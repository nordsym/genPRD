## Section 0: Agent Brief
Build a project management tool that becomes the heartbeat of small teams. Teams of 3-10 need to seamlessly coordinate, share updates, and track progress without getting bogged down in meetings or emails. Your mission: create an intuitive, real-time platform where tasks are effortlessly managed, progress is visible, and collaboration is second nature. Priority: fluid real-time collaboration > intuitive task management > sleek, minimal design.

## Section 1: Overview
**What:** A real-time project management tool tailored for small teams.
**Who:** Small teams, especially in startups and small businesses.
**Why:** To facilitate seamless coordination and ensure everyone is on the same page without constant meetings.
**Success:** A team of 6 can launch a new campaign by creating a project, assigning tasks, and tracking progress in real time.

## Section 2: Pages & Navigation
**Pages:**
- `/login` - Email/password entry (public)
- `/signup` - Account creation flow (public)
- `/dashboard` - Overview of active projects and personal tasks (auth required)
- `/projects` - List view of all team projects (auth required)
- `/projects/:id` - Kanban board and task details (auth required)
- `/team` - Member management and presence view (auth required)
- `/settings` - User profile and notifications (auth required)

**Navigation:**
- Sidebar (desktop): Dashboard, Projects, My Tasks, Team, Settings
- Mobile: Bottom tab bar for key views
- Auth gate: Redirect to `/login` if unauthenticated

## Section 3: Core Features

### Feature: Project & Task Management
**What user does:**
- Creates projects with title/description.
- Adds tasks with assignee, due date, and priority.
- Drags tasks between "To Do", "In Progress", "Done".

**What user sees:**
- Kanban board with drag-and-drop columns.
- "Create Task" modal with rich text description.

**Must work:**
- Dragging a task updates status instantly for all online users.
- Task counts update in real-time on dashboard.

### Feature: Real-time Collaboration
**What user does:**
- Sees who is viewing the same project.
- Receives toast notifications when assigned a task.

**What user sees:**
- Avatar pile (facepile) showing online members.
- Live typing indicators in comments.

**Edge cases:**
- Offline: Show "Reconnecting..." badge.
- Conflict: Last-write-wins strategy for task descriptions.

## Section 4: Data Architecture

**Table: users**
- `id`: uuid PRIMARY KEY
- `email`: text UNIQUE NOT NULL
- `name`: text
- `avatar_url`: text
- `last_seen_at`: timestamp

**Table: projects**
- `id`: uuid PRIMARY KEY
- `team_id`: uuid REFERENCES teams(id)
- `title`: text NOT NULL
- `status`: text DEFAULT 'active'
- `created_at`: timestamp DEFAULT now()

**Table: tasks**
- `id`: uuid PRIMARY KEY
- `project_id`: uuid REFERENCES projects(id) ON DELETE CASCADE
- `assignee_id`: uuid REFERENCES users(id)
- `status`: text DEFAULT 'todo' (todo, in_progress, done)
- `position`: integer (for sorting)
- `due_date`: timestamp

**RLS Policies:**
- `projects`: Users can view/edit projects belonging to their `team_id`.
- `tasks`: Inherit permissions from the parent project.

## Section 5: Technical Approach
- **Real-time:** Use Supabase Realtime (Postgres Changes) to listen for task updates and cursor positions.
- **State Management:** Use optimistic UI updates for drag-and-drop to ensure 60fps smoothness even with latency.
- **Auth:** Standard email/password flow with persistent sessions.

## Section 6: Visual Design
- **Style:** Clean and minimalistic (think Linear or Notion).
- **Colors:**
  - Primary: Deep Indigo (#4F46E5)
  - Statuses: Todo (Gray), In Progress (Blue), Done (Green)
- **Layout:** Desktop-first, collapsible sidebar to maximize board space.
